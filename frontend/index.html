Startup Flow
  1. Create the startup and founder admin in one request.
  2. Verify the founder account.
  3. Log in with the startup slug.
  4. Open the startup dashboard and manage team members.

What changed

The startup name and the startup slug are now treated differently. Founders create a startup by name, Sentinel provisions an internal slug like codearena, and every future login for that startup uses the slug behind the scenes.

Startup Name

The human-facing company name like CodeArena or Acme Cloud.

Startup Slug

The machine-safe identifier like codearena that routes tenant-scoped auth requests.

Founder Admin

The first account is created as an admin for that startup and verifies before logging in.
